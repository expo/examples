# Add Legend-State with Supabase Sync

## When to use

- User wants offline-first state management with cloud sync
- User asks about Legend-State or Legend-State with Supabase
- User needs real-time data synchronization backed by Supabase/Postgres
- User wants reactive observable state that persists locally and syncs remotely

## Dependencies

```bash
npm install @legendapp/state @supabase/supabase-js @react-native-async-storage/async-storage react-native-get-random-values uuid
npx expo install react-native-safe-area-context react-native-reanimated
```

Dev dependencies:

```bash
npm install --save-dev @types/uuid
```

## Configuration

### Environment variables

Create or update `.env`:

```
EXPO_PUBLIC_SUPABASE_URL=https://<your-project>.supabase.co
EXPO_PUBLIC_SUPABASE_ANON_KEY=<your-anon-key>
```

### Supabase table setup

Create a table with timestamp and soft-delete columns for incremental sync:

```sql
create table todos (
  id uuid default gen_random_uuid() primary key,
  counter bigint generated by default as identity,
  text text,
  done boolean default false,
  created_at timestamptz default now(),
  updated_at timestamptz default now(),
  deleted boolean default false
);

alter publication supabase_realtime add table todos;
```

## Implementation

### 1. Create the Supabase + Legend-State sync module

Create `utils/SupaLegend.ts`:

```tsx
import { createClient } from "@supabase/supabase-js";
import { observable } from "@legendapp/state";
import { syncedSupabase } from "@legendapp/state/sync-plugins/supabase";
import { configureSynced } from "@legendapp/state/sync";
import { observablePersistAsyncStorage } from "@legendapp/state/persist-plugins/async-storage";
import AsyncStorage from "@react-native-async-storage/async-storage";
import "react-native-get-random-values";
import { v4 as uuidv4 } from "uuid";

const supabase = createClient(
  process.env.EXPO_PUBLIC_SUPABASE_URL,
  process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY
);

const generateId = () => uuidv4();

const customSynced = configureSynced(syncedSupabase, {
  persist: {
    plugin: observablePersistAsyncStorage({ AsyncStorage }),
  },
  generateId,
  supabase,
  changesSince: "last-sync",
  fieldCreatedAt: "created_at",
  fieldUpdatedAt: "updated_at",
  fieldDeleted: "deleted",
});

export const todos$ = observable(
  customSynced({
    supabase,
    collection: "todos",
    select: (from) =>
      from.select("id,counter,text,done,created_at,updated_at,deleted"),
    actions: ["read", "create", "update", "delete"],
    realtime: true,
    persist: { name: "todos", retrySync: true },
    retry: { infinite: true },
  })
);

export function addTodo(text: string) {
  const id = generateId();
  todos$[id].assign({ id, text });
}

export function toggleDone(id: string) {
  todos$[id].done.set((prev) => !prev);
}
```

### 2. Use in components

Wrap reactive components with `observer`:

```tsx
import { observer } from "@legendapp/state/react";
import { todos$, addTodo, toggleDone } from "./utils/SupaLegend";

const Todos = observer(() => {
  const todos = todos$.get();
  if (!todos) return null;
  return (
    <FlatList
      data={Object.values(todos)}
      renderItem={({ item }) => (
        <TouchableOpacity onPress={() => toggleDone(item.id)}>
          <Text>{item.text}</Text>
        </TouchableOpacity>
      )}
    />
  );
});
```

## Key API reference

| API | Purpose |
|-----|---------|
| `observable(customSynced({...}))` | Create an observable synced to a Supabase table |
| `configureSynced(syncedSupabase, opts)` | Create a reusable sync config with shared defaults |
| `observer(Component)` | HOC that re-renders when observed observables change |
| `obs$.get()` | Read current value and subscribe to updates |
| `obs$[id].assign({...})` | Assign fields to trigger a create/update |
| `obs$[id].field.set(value)` | Update a single field, synced to Supabase |

## Adaptation notes

- Merge dependencies — do not replace `package.json`
- Set `EXPO_PUBLIC_SUPABASE_URL` and `EXPO_PUBLIC_SUPABASE_ANON_KEY` in your environment
- Adapt the SQL migration to your data model (table names, columns)
- Add `created_at`, `updated_at`, and `deleted` columns to every synced table
- Enable realtime on each table with `alter publication supabase_realtime add table <name>`
- The `configureSynced` wrapper can be reused across multiple observables for different tables
- Import `react-native-get-random-values` before `uuid` to polyfill crypto on React Native
